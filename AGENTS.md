# Repository Guidelines

## Purpose

`zsh-codex-mode` turns the current Zsh line editor into a streaming Codex conversation. Keep the implementation small and Zsh-native. The runtime dependencies are Zsh, `jq`, and Codex CLI; do not reintroduce the previous Python or Zig implementations.

## Structure

- `zsh-codex-mode.plugin.zsh`: the entire runtime plugin
- `README.md`: installation, configuration, and user-facing behavior
- `demo/`: deterministic demo build
  - `build.zsh`: creates the fixture and builds `assets/demo.gif`
  - `demo.tape`: VHS presentation and timing
  - `responses.jsonl`: recorded app-server messages and delays
  - `codex`: local app-server replay shim; it must not invoke real Codex
  - `fixture/` and `changes.patch`: demo repository and task
- `assets/demo.gif`: generated by `demo/build.zsh`

The Home Manager consumer is in `/home/yifan/nix/home-manager/shell.nix`. It pins this repository with `fetchFromGitHub` and explicitly sets only `ZSH_CODEX_MODE_MCP=disabled`.

## Behavioral Invariants

- Ctrl-X enters and leaves Codex mode. Ctrl-D must leave Codex mode without exiting the shell.
- Entering the mode must show the Codex prompt immediately. App-server initialization may happen lazily on the first message.
- One mode session owns one app-server process and one thread ID. Follow-up turns reuse both. Leaving the mode destroys the process and conversation.
- Codex mode shows only `ZSH_CODEX_MODE_PROMPT` (default `✨ `), not the normal shell prompt.
- Messages stream without losing the final line or duplicating the prompt. Preserve terminal newlines carefully; this area has regressed before.
- User input uses `ZSH_CODEX_MODE_INPUT_STYLE` (default `fg=green`). An empty value disables the style.
- zsh-autosuggestions and zsh-syntax-highlighting pause while the mode is active and are restored on exit. zsh-vi-mode must remain supported.
- Tool activity is a short one-line hint. Collapse whitespace, respect `ZSH_CODEX_MODE_ACTIVITY_MAX_LENGTH`, and do not print command output or reasoning.
- Model, reasoning effort, sandbox, approval policy, and MCP inherit Codex settings unless explicitly overridden.
- Codex plugins are disabled for the app-server process.

## Implementation Notes

- Codex app-server communicates as JSONL over a Zsh coprocess using `--stdio`.
- Build JSON with `jq`; do not hand-roll JSON escaping in Zsh.
- `initialize`, `config/read`, and `thread/start` happen once per mode session. Each prompt sends only `turn/start` afterward.
- ZLE behavior must be tested in a real interactive PTY. Non-interactive `zsh -c` tests cannot prove prompt, redraw, keymap, or Ctrl-D behavior.
- Key bindings may differ across `main`, `emacs`, `viins`, and `vicmd`. Inspect the active keymap when debugging zsh-vi-mode interactions.

## Current Investigation

Ctrl-D has been observed to exit the shell even after entering Codex mode. A minimal interactive reproduction also exits. Do not assume that binding `^D` in only `emacs` and `viins` is sufficient; determine whether the active keymap, ZLE EOF handling, or server shutdown is responsible, and verify the fix in a PTY.

## Validation

Run the focused checks after plugin changes:

```sh
zsh -n zsh-codex-mode.plugin.zsh demo/build.zsh demo/codex
jq -e . demo/responses.jsonl >/dev/null
git diff --check
```

For demo changes, install VHS, jq, Git, Node.js, and Zsh, then run:

```sh
demo/build.zsh
```

The demo must remain deterministic in content and timing and must not use a Codex account, network request, or live model inference.

## Change Discipline

- Prefer direct code over speculative helpers, compatibility layers, fallbacks, or feature flags.
- Keep README prose concise. Enumerate configuration values when the option has a fixed set.
- Do not commit or push unless the user explicitly asks.
- When updating the Home Manager pin, use the full commit hash, recompute the fixed-output hash, run `just fmt` and `just check` in `/home/yifan/nix`, and apply with `just home` only when requested.
